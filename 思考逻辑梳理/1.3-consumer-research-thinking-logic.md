## 一、决策演进树

```
起点：1.1（行业）与 1.2（竞争）已完成，需要把“人群洞察”转为可执行决策接口
  ↓
核心问题 1：人群细分用什么主逻辑，才能服务后续定位与机会判断？
  → 仅用人口统计：可解释弱、对策略动作映射差
  → 决策：采用“决策驱动主轴”作为第一切分维度
  ↓
核心问题 2：主轴选择如何避免主观拍脑袋？
  → 主观区间打分：不同模型/执行者波动大
  → 决策：二元检查清单（0/1）+ 加权计算（40/35/25）
  ↓
核心问题 3：单一主轴会不会遗漏关键行为差异？
  → 决策：叠加强制覆盖层（价格敏感度/品牌忠诚度/渠道偏好）
  ↓
核心问题 4：字段填充如何防止“画像编造”？
  → 决策：证据门控（有证据才填；无证据统一标注 unknown/insufficient evidence）
  ↓
核心问题 5：与 1.1/1.2 是替代关系还是继承关系？
  → 决策：上游数据融合验证（默认信任 + 一致性检查 + 冲突分级处理）
  ↓
核心问题 6：边界场景怎么处理（平分、数据不足、B2B）？
  → 决策：tie-break + 降级模式 + B2B 映射规则
  ↓
核心问题 7：质量保障如何“可执行”而非口号？
  → 决策：P0/P1/P2 三层检查（P0 阻塞项保障决策安全）
  ↓
核心问题 8：如何保证该模块可被后续模块直接消费？
  → 决策：5 层 JSON Schema + 下游交接包（1.4/1.5/2.1）
  ↓
质量收敛：Momus 6 轮审查后，形成“主轴选择—证据门控—下游交接”闭环
```

## 二、设计原则演进

### 原则 1：决策可执行性优先于描述完整性

**触发事件**：传统画像能描述用户，但难以直接转译成定位、产品、渠道动作。  
**原则落地**：主轴必须能映射到策略动作，优先保证“可用”而非“好看”。

### 原则 2：确定性优先于“智能感”

**触发事件**：主观区间评分在不同执行上下文下不稳定。  
**原则落地**：主轴改为二元检查清单 + 明确权重，减少解释漂移。

### 原则 3：证据优先于推断

**触发事件**：消费者字段最容易被模型补全。  
**原则落地**：证据门控作为强约束；没有证据就显式标注不足，不允许隐式猜测。

### 原则 4：继承上游、避免重复建设

**触发事件**：1.3 若重做 1.1/1.2，会增加冲突与冗余。  
**原则落地**：建立信任链与冲突协议，只在必要处做补证和校验。

### 原则 5：边界条件先定义，避免执行时补丁

**触发事件**：平分主轴、外部数据失败、B2B 场景都会导致流程歧义。  
**原则落地**：在 SOP 内前置 tie-break、降级模式、B2B 映射，保证可连续执行。

## 三、关键决策点

| 决策点 | 备选 | 最终选择 | 核心理由 |
|---|---|---|---|
| 人群细分主逻辑 | 人口统计优先 / 决策驱动 | 决策驱动主轴 | 直接连接下游策略动作 |
| 主轴评分机制 | 主观区间 / 二元检查清单 | 二元检查清单 + 加权 | 可重复、可审计、波动更小 |
| 主轴不足补偿 | 无覆盖层 / 强制覆盖层 | 强制覆盖层 | 防止单轴遗漏关键行为差异 |
| 字段填充标准 | 允许经验补全 / 证据门控 | 证据门控 | 降低画像幻觉风险 |
| 上游关系 | 重建上游 / 融合验证 | 融合验证 | 降低重复劳动并保留一致性 |
| 异常场景处理 | 临场判断 / 预置规则 | tie-break + 降级 + B2B 适配 | 保证执行连续性 |
| 质量保障 | 单层检查 / 三层检查 | P0/P1/P2 | 先决策安全，再价值增强 |
| 输出接口 | 文本输出 / 结构化输出 | 5 层 JSON + downstream_handoff | 支持模块化自动编排 |

## 四、自我纠错记录

| 问题/风险 | 发现方式 | 修正动作 |
|---|---|---|
| 主轴选择存在主观波动 | 方案评审与执行一致性检查 | 改为二元检查清单（0/1）+ 固定权重 |
| 画像字段可能被“合理补全” | 质量门控审查 | 引入证据门控与 unknown/insufficient evidence 标记 |
| 1.2 口碑数据可用性被默认化 | 依赖关系复核 | 增加条件可用状态与降级路径 |
| 平分主轴导致决策卡顿 | 规则演练 | 增加 tie-break 顺序与混合模式 |
| 仅面向 B2C，泛化性不足 | 边界场景审查 | 增加 B2B（DMU）映射规则 |
| 输出难以直接供下游消费 | 接口审查 | 固化 5 层 Schema 与下游交接字段 |

## 五、版本演进总结

| 版本 | 核心能力 | 价值 |
|---|---|---|
| **v3.0（当前）** | 决策驱动主轴、二元检查清单、证据门控、融合验证、P0/P1/P2、5层JSON接口 | 将“消费者研究”从报告型输出升级为“可编排决策模块” |

**本次沉淀结论**：1.3 的关键不在“多写画像字段”，而在把“谁值得优先服务、为什么、如何交给下游执行”变成确定性流程与结构化接口。
