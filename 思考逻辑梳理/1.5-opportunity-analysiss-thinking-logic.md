## 2. 本章节当前要解决的核心问题（不变）

### 2.1 当前要解决的核心问题
1.5 不是做新的研究模块，而是把 1.2 / 1.3 / 1.4 的上游信号整合成可筛选、可排序、可交接的“机会判断模块”。

当前核心问题集中在三层：
1. **机会项如何标准化表达**
2. **机会类型如何稳定分类**
3. **机会是否足以进入优先级排序，如何判断**

### 2.2 到目前为止推进到哪些子问题（不变）

### 3.2 过程中出现过哪些方案、分歧、质疑、修正或推翻（增量更新）
（保留原有方案一至方案四，新增以下三条）：

#### 方案五：重复机会采用"3/4 相同"即可判定
- 提出：在去重规则中使用"目标人群、核心场景、问题缺口、动作方向四项中三项相同视为重复"。
- 被质疑：阈值"3"缺乏项目内依据，且未考虑`action_direction`在机会判定中的核心权重。
- 修正：改为**主判定字段全部相同 + 辅判定字段至少满足 1 项**的复合规则。主判定字段为 `target_segment` + `core_scenario` + `action_direction`，辅判定字段为 `gap_description` / 证据指向 / `opportunity_type` 中至少一致一项。

#### 方案六：使用"核心对象词/动作目标词"进行证据链验证
- 提出：要求"已观察事实与问题缺口至少共享 1 个核心对象词，问题缺口与动作方向至少共享 1 个动作目标词"。
- 被质疑：AI难以稳定判定何为"核心对象词"、何为"动作目标词"，属于新引入的未定义模糊术语。
- 修正：删除"核心对象词""动作目标词"的说法，直接依据固定字段（`target_segment`/`core_scenario`/`problem_object`/`action_object`）做一致或差异判断；若字段无法对齐，则判定为证据链不完整。

#### 方案七：沿用"资源等级/资源挤占"等资源类判断词
- 提出：在"竞争可切入性""战略承接性"中使用"所需资源等级""资源挤占标记"等表述。
- 查证发现：仓库内无任何既定义或明确定义，且与 5.3 上下文无法闭合定义。
- 结论：这些词在 5.3 中属于新的模糊引入，必须整体移除。
- 替代方向：转用 5.3 自身可闭合定义的字段，如：新增前置条件数（`new_prerequisite_count`）、新增假设数（`new_assumption_count`）、竞争封锁标记（`competition_block_flag`）。

## 5. 关键决策点

### 5.1 目前已相对明确的关键决策（增补）
（保留决策 1–5 不变）

#### 决策 6：机会筛选核心字段的一一对应规则
- **结论**：证据链成立的判定不依赖语义共享，而依赖以下显式字段：
  - `target_segment`（目标人群）
  - `core_scenario`（核心场景）
  - `problem_object`（问题对象）
  - `action_object`（动作对象）
- 判定条件围绕这些字段的一致性与可映射性展开；若无法满足一致性或可映射，则判定为证据链不完整。

#### 决策 7：去重规则的核心依据
- **结论**：去重不依据"N/4 相同"，而依据：
  - 主判定字段（`target_segment` + `core_scenario` + `action_direction`）全部一致
  - 辅判定字段（`gap_description` / 证据指向 / `opportunity_type`）至少一致一项
- 优先级：证据条数更多者优先；若相同则证据类型数更多者优先；若仍相同则 opportunity_id 较小者优先。

## 6. 自我纠错记录

### 6.1 中间出现过哪些错误判断、跳步、遗漏或表达偏差（保留）
1. **把"适用场景""关键词""示例"写得太像枚举**——容易误导为封闭列表。
2. **"组合机会"边界初稿偏宽**——容易被误用为兜底分类。
3. **筛选标准中出现较多抽象词**——如"明显跳跃""不足以支持""增长迹象""能形成启发"等。
4. **曾倾向于把机会筛选理解成语义判断**——但这与项目现有"字段化、阈值化、质检化"的方法不一致。
5. **"强证据/弱证据"初稿未接到已有置信度体系**——造成可执行性不足。

### 6.2 这些问题是如何被识别并修正的（保留）
1. 通过用户追问"是否只是列举不是枚举"，发现组合机会描述需要更严格的开放式写法。
2. 通过用户追问"其他机会是否也有同样问题"，发现六类机会都应统一处理为"定义固定、表现开放"。
3. 通过用户对筛选规则可执行性的质疑，发现多处自然语言判断无法直接由 AI 稳定执行。
4. 结合仓库中 1.2、1.4、冲突处理规则的写法，确认本项目的成熟范式是：用明确字段代替模糊术语；用阈值代替感觉；用分支规则代替笼统判断。

### 6.3 本轮新增的误用与偏差（新增）
1. **误用"3/4"作为去重阈值**
   - 问题：数值缺乏 rationale，且未考虑 `action_direction` 的决定性权重。
   - 识别：用户质疑"为什么 3/4"。
   - 修正：改用主+辅判定字段组合。

2. **沿用资源类术语导致新模糊点引入**
   - 问题：将"资源等级/资源挤占/关键能力/主计划"等词带入 5.3。
   - 识别：用户要求查证其在仓库中的定义；发现无命中。
   - 修正：删除整个资源判断框架，改用前置条件/假设计数方式替代。

3. **在证据链验证中引入"核心对象词/动作目标词"**
   - 问题：语义类未定义术语，AI 无法稳定复核。
   - 识别：用户追问"核心对象词指什么"。
   - 修正：删除该说法，直接规定依据哪些字段做一致性判定。

### 6.4 本轮发生的范式强化（新增）
1. **反模糊原则进一步具象**
   - 不允许新增任何仓库内未定义的术语；若目标功能无法用现有字段表达，则应扩展字段定义而非引入新词。
   - 所有判断必须落到 0/1/2 分、阈值比较、枚举比对、计数 ≥ N 的形式。

2. **质量检查的禁止词列表扩容**
   - 将"核心/主线/关键/主计划/资源等级/资源挤占/假设"等全部加入"不得出现"清单，在最终质量检查中逐项排除。
