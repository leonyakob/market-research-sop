# 品牌策划全案 SOP 项目规则

> 本项目特定规则，优先级高于全局 AGENTS.md。
> 通用规则请参考：~/.config/opencode/AGENTS.md

---

## 一、项目定义

本项目为品牌策划全案 SOP 构建项目，目标是将品牌策划流程沉淀为可迭代、可复用、可版本管理的结构化方法论体系。

项目采用：模块化结构 + 版本化迭代机制 + GitHub 持久化存储

---

## 二、结构规则（Structure Rules）

1. 项目按"模块 → 子模块 → SOP"进行分层管理，例如：

   ```
   市场调研 / 行业研究 / SOP
   ```

2. 每个子模块为一个独立单元（如"行业研究"），具备独立的版本演进历史；

3. 每个子模块目录下必须包含：

   - 多版本 SOP 文件（v1, v2, v3…）
   - 一个 README.md 用于记录版本说明

---

## 三、版本管理规则（Versioning Rules）

1. 所有 SOP 优化必须遵循：

   **不允许覆盖旧版本，只能创建新版本**

2. 版本命名规范：

   - v1_行业研究SOP.md
   - v2_行业研究SOP_优化版.md
   - v3_行业研究SOP_结构重构版.md

3. 每个版本必须：

   - 保留完整内容（不可只写增量）
   - 明确版本语义（优化点/调整方向）

4. 每次版本更新后：

    自动提交（commit）并同步至 GitHub 仓库进行长期存档

---

## 四、README 机制（版本说明系统）

每个子模块必须维护一个 README.md 文件，用于记录版本演进逻辑。

### README 标准结构：

```markdown
# 行业研究 SOP 版本说明

## v1（初版）
初版流程，覆盖基础行业分析逻辑

## v2（优化版）
优化点：
- 增加竞品分析维度
- 调整信息收集路径
- 明确输出格式

## v3（结构重构版）
优化点：
- 重构分析框架
- 提升模块复用性
```

---

## 五、行为规则（AI执行逻辑）

### 1. 当用户说："优化当前 SOP"

系统必须执行：

- 基于当前版本生成新版本（+1）
- 保留原版本不变
- 输出完整新 SOP（非增量修改）
- 自动更新 README.md：
  - 新增该版本记录
  - 总结优化点
- 自动执行 Git 提交并推送

---

### 2. 当用户说："回到某个版本（如 v1 / 行业研究初版）"

系统必须执行：

- 定位对应版本文件
- 将该版本内容恢复为当前工作内容
- 不删除其他版本文件
- 标记当前状态为"回溯使用版本"

---

### 3. 当用户说："完成一个阶段 / 保存当前版本"

系统必须执行：

- 将当前内容整理为一个新版本（如 v3）
- 自动生成版本命名（基于语义）
- 更新 README.md
- 执行 GitHub 提交

---

## 六、GitHub 同步规则

所有版本操作必须同步到 GitHub 仓库：

- 每次新增版本 → commit
- commit message 必须包含：
  - 模块名
  - 版本号
  - 变更摘要

示例：

```
feat: 行业研究SOP v2（竞品分析增强 + 流程优化）
```

---

## 七、核心设计原则

1. 版本不是覆盖，而是资产积累
2. 每一个 SOP 都是可复用模块
3. README 是方法论演进记录，而不是备注
4. GitHub 是"知识资产库"，不是代码仓库
5. 数据驱动：所有分析结论必须有数据支撑，数据来源必须可追溯

---

## 八、一句话总结

本系统是一个：基于 GitHub 的品牌策划方法论版本管理系统（Versioned SOP System）

## 九、SOP v3.0 执行规则

### 9.1 数据获取
- 所有SOP模块的数据获取必须通过 `.sop/config/data-acquisition.yaml` 配置
- 当前使用 Exa/Tavily 外部搜索，未来切换为自建数据源（自然语言API）
- 切换方式：修改配置文件中的 `primary` 字段，SOP文件无需改动
- 数据调用使用自然语言查询模板，非固定搜索词

### 9.2 数据融合
- 外部数据与内部数据是"融合验证"关系，非"覆盖替代"
- 详见 `交付文档/SOP_v3.0_融合与冲突处理规则.md`

### 9.3 内部文件处理
- 内部文件标注只写文件名，不写路径（涉密）
- 详见 `.sop/config/internal-files.yaml`

### 9.4 模板使用
- 所有SOP模块必须基于 `交付文档/SOP_v3.0_通用模板.md` 的骨架结构创建
- 模板仅定义章节骨架（触发条件→输入参数→数据获取→输出格式→分析规则→质量检查），具体内容由各模块自定义
