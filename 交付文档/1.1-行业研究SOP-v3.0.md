# 1.1 行业研究 SOP框架 v3.0

> **模板说明**：本文档为1.1行业研究模块的v3.0版本，基于SOP v3.0通用模板构建。
> 核心升级：数据调用策略（支持热切换）、双通道数据融合、内部文件接口、P0/P1/P2分级检查、默认值机制。

---

## 前置声明

### "求真务实"原则（5条规则）

AI在执行本SOP时，**必须严格遵循以下规则**：

1. **所有事实性信息必须来源于可验证的资料**（如学术文献、官方文档、权威出版物、可靠数据库、政府统计、上市公司财报）
2. **每一条关键结论应附带来源说明或推理依据**
3. **禁止生成无依据的推测、臆断或虚构内容**
4. **当缺乏足够信息支持时，应明确说明"信息不足"，而非补全或猜测**
5. **如存在多种观点，应说明不同来源及其差异，而非合并或简化为单一结论**

### 本模块在行业研究中的位置

```
市场调研
├── 1.1 行业研究 ← [本模块]
├── 1.2 竞争分析
├── 1.3 消费者研究
├── 1.4 趋势分析
└── 1.5 机会分析
```

### 与其他模块的依赖关系

| 依赖模块 | 关系类型 | 说明 |
|----------|----------|------|
| 无 | — | 本模块为市场调研首个模块，无前置依赖 |
| 1.2 竞争分析 | 输出依赖 | 本模块输出的行业规模、生命周期作为竞争分析输入 |
| 1.3 消费者研究 | 输出依赖 | 本模块PEST社会文化维度作为消费者研究背景 |
| 2.1 市场定位 | 输出依赖 | 本模块输出的机会与风险作为市场定位依据 |

### 本模块结构概览

本报告包含9个部分（PEST分析、行业规模与增长、细分市场分析、行业生命周期、价值链分析、竞争格局、关键成功因素、机会与风险、战略建议）+ 4张数据表。

需要数据：宏观政策、市场规模、品牌份额、消费人群、功能性成分、产业链等。

## 一、触发条件

### 1.1 何时执行本模块

本模块在以下场景触发执行：

- **新品牌进入**：计划进入新品类市场，需系统性了解行业全貌
- **产品线规划**：现有品牌计划扩展新品类，需评估市场机会
- **投资评估**：对某品类进行投资可行性分析
- **战略调整**：现有业务方向调整，需重新评估行业环境

### 1.2 前置模块完成状态

本模块为市场调研首个模块，无前置模块。可直接执行。

## 三、数据调用策略

> **数据源配置**：`.sop/config/data-acquisition.yaml`
> 
> **当前状态**：Exa/Tavily 外部搜索（网页内容 → AI提取数据）
> 
> **未来状态**：自建数据源 API（自然语言查询 → 结构化JSON → AI直接使用）
> 
> **切换方式**：修改配置文件中的 `primary` 字段，**SOP无需任何改动**

### 3.1 数据调用指令

**配置引用**：`.sop/config/data-acquisition.yaml`

**执行逻辑**：

```
AI生成自然语言查询 → 调用主数据源（当前Exa，未来internal_api）
  → 若失败 → 自动降级fallback（Tavily）
  → 返回数据 → AI提取洞察、填入报告
```

**数据调用指令模板**（AI按此格式生成查询）：

| 数据维度 | 自然语言查询模板 | 预期返回内容 |
|---------|----------------|-------------|
| **市场规模** | `获取{品类}在{地理范围}的市场规模历史数据（过去{N}年）及预测（未来{N}年），包含各年份数值、单位、增长率` | 市场规模时间序列 + CAGR + 数据口径说明 |
| **竞争格局** | `获取{品类}在{地理范围}的主要品牌市场份额排名、CR5/CR10数据、品牌梯队划分` | 品牌份额排名 + CR5 + 梯队分析 |
| **消费人群** | `获取{品类}在{地理范围}的消费人群画像，包含年龄分布、性别比例、收入水平、消费场景` | 人群画像数据 + 消费场景描述 |
| **行业政策** | `获取{品类}在{地理范围}的行业政策、监管法规、国家标准、市场准入要求` | 政策清单 + 法规要点 + 准入条件 |
| **功能性成分** | `获取{成分}在{应用场景}的市场规模、增速、主要供应商、应用领域` | 成分市场数据 + 供应商 + 应用场景 |
| **行业生命周期** | `获取{品类}在{地理范围}的行业渗透率、品牌数量变化、增长率趋势，用于判断行业生命周期阶段` | 渗透率 + 品牌数量趋势 + 增长率 |
| **价值链** | `获取{品类}产业链结构，包含上游原料、中游生产、下游渠道各环节的成本占比、利润分配、关键参与者` | 产业链结构 + 各环节利润 + 参与者 |

**占位符替换规则**：
- `{品类}` → 用户输入的目标品类（如"无糖功能性茶饮"）
- `{地理范围}` → 确认的地理范围（如"中国"）
- `{N}` → 确认的时间范围年数
- `{成分}` → 功能方向相关成分（如"GABA"、"茶氨酸"）
- `{应用场景}` → 功能方向（如"舒缓"、"提神"）

### 3.2 当前阶段：Exa/Tavily 搜索

**当前主数据源**：Exa（`primary: exa`）
**降级数据源**：Tavily（`fallback: tavily`）

**返回格式（当前）**：

```json
{
  "results": [
    {
      "title": "页面标题",
      "url": "https://...",
      "publishedDate": "2024-01-15",
      "text": "网页完整文本内容...",
      "highlights": ["AI提取的关键片段1", "关键片段2"],
      "summary": "AI生成的页面摘要"
    }
  ]
}
```

**AI处理方式**：
1. 从返回的网页文本/highlights/summary中**提取**关键数据（数值、单位、年份、来源）
2. 交叉验证多个结果的一致性
3. 按来源标注规范记录数据来源
4. 将提取的数据填入报告

### 3.3 未来阶段：自建数据源 API

**未来主数据源**：自建数据源（`primary: internal_api`）

**返回格式（预期）**：

```json
{
  "query": "原始查询",
  "answer": "综合回答（整合多源后的分析性文本，包含趋势说明、口径说明、竞争格局背景）",
  "metrics": [
    {"name": "指标名称", "value": 数值, "unit": "单位", "year": 年份, "confidence": "高/中/低"}
  ],
  "sources": [
    {"name": "来源名称", "type": "来源类型", "url": "链接"}
  ],
  "notes": "补充说明（口径差异、数据冲突处理说明等）"
}
```

**AI处理方式**：
1. `metrics` → 直接填入数据表
2. `answer` → 理解背景，提取洞察写入分析段落
3. `notes` → 在报告中标注口径说明
4. `sources` → 生成引用标注

**切换步骤**（技术人员执行）：
1. 部署自建数据源API
2. 配置 `.sop/config/data-acquisition.yaml` 中的 `endpoint` 和 `auth_token`
3. 修改 `primary: internal_api`
4. 测试验证
5. **SOP文件无需任何修改**

### 3.4 内部文件处理

**配置文件引用**：`.sop/config/internal-files.yaml`

**文件读取规则**：

| 文件大类 | 读取方式 | 标注规范 | 隐私级别 |
|----------|----------|----------|---------|
| 市场与销售类 | 数据清洗→字段标准化→样本校验 | 【内部-市场数据-文件名】 | 中/高 |
| 竞品与情报类 | 来源标注→时效性检查→交叉验证 | 【内部-竞品情报-文件名】 | 中/高 |
| 产品与供应链类 | 成本拆分→工艺标准化→风险标注 | 【内部-供应链-文件名】 | 高 |
| 财务与战略类 | 高度脱敏→比例保留→假设标注 | 【内部-财务-文件名】 | 高 |
| 组织与运营类 | 信息脱敏→经验提炼→数据保留 | 【内部-运营-文件名】 | 低/中 |

**内部文件标注规范**：

```
【内部-大类-子类_{标识}_{日期}.{格式}】
- 示例：【内部-市场数据-销售数据_华东渠道_Q4_2024.xlsx】
- 示例：【内部-竞品情报-价格监控_东方树叶_202412.xlsx】
```

**融合规则**：内部数据与外部数据并行比对，差异<5%标注"已验证"，差异>10%触发冲突预警。详见 `SOP_v3.0_融合与冲突处理规则.md`。

### 3.5 AI自主补充查询

**补充触发条件表**：

| 条件 | 判断方法 | 阈值 | 补充查询指令 |
|------|----------|------|-------------|
| 政策相关度高 | 政策维度返回内容中"政策/法规/标准"出现≥3次 | 高频 | 补充查询：行业准入门槛 + 监管趋势 |
| 市场规模数据不足 | 数据调用未返回具体数值 | 无数据 | 补充查询：行业报告 + 市场研究 + 咨询机构 |
| 消费者画像模糊 | 未返回年龄/性别/收入数据 | 缺失 | 补充查询：用户画像 + 消费人群 + 调研报告 |
| 技术关键词高频 | 返回内容中"技术/创新/专利"出现≥3次 | 高频 | 补充查询：技术壁垒 + 研发投入 + 专利布局 |
| 数据冲突 | 多源数据差异 > 10% | >10% | 补充查询：第三方验证数据 + 统计口径说明 |

**数据调用失败处理流程**：

```
主数据源调用失败 → 自动降级fallback（Tavily）
  → 同义词扩展查询（如"市场规模"→"市场容量"→"市场空间"）
  → 扩大查询范围（如"无糖茶"→"茶饮料"→"软饮"）
  → 若仍无结果 → 标注"信息不足"并说明已尝试路径
  → 继续执行其他维度，不阻塞流程
```

## 四、输出格式

### 4.1 行业概况报告结构（9个部分）

**一、PEST分析**
- 1.1 政治政策环境：核心发现、**证据来源**、对行业影响
- 1.2 经济环境：（同上结构）
- 1.3 社会文化环境：（同上结构）
- 1.4 技术环境：（同上结构）

**二、行业规模与增长**
- 2.1 市场规模：数据[数值+单位+年份+**来源标注**]、趋势
- 2.2 增长率：历史增长率[**来源标注**]、预测增长率[**来源标注**]

**三、细分市场分析（如适用）**
- 3.1 目标品类细分：各细分市场份额[**来源标注**]
- 3.2 功能性成分市场：相关成分市场规模+增速[**来源标注**]

**四、行业生命周期**
- 4.1 当前阶段判断：[导入/成长/成熟/衰退]
- 4.2 判断依据：[证据列表，**每条需来源**]
- 4.3 发展趋势预判：[分析，**需有数据支撑**]

**五、价值链分析**
- 5.1 产业链结构：[图示或描述]
- 5.2 各环节价值分配：[数据，**需来源**]
- 5.3 关键参与者：[列表]

**六、竞争格局（波特五力）**
- 6.1 五力强度评估
  | 五力维度 | 强度 | 证据来源（**必须具体，包含数据来源和分析依据**） |
  |----------|------|--------------------------------------------------|
  | 现有竞争者 | 高/中/低 | **示例**：根据XXX报告，CR5为XX%，属于高强度竞争 |
  | 潜在进入者 | 高/中/低 | **示例**：根据YYY分析，进入壁垒为XXXX |
  | 替代品威胁 | 高/中/低 | **示例**：根据ZZZ分析，替代品威胁为XXXX |
  | 供应商议价 | 高/中/低 | **示例**：XXXX |
  | 购买者议价 | 高/中/低 | **示例**：XXXX |
- 6.2 竞争格局概览
  - 主要参与者：[列表]
  - 份额：（**必须标注来源**）
  - 品牌梯队：[分析，**需有数据支撑**]

**七、关键成功因素 (KSFs)**
- 7.1 KSF清单
  | 因素 | 重要性 | 行业表现 | 证据（**必须具体**） |
  |------|--------|----------|---------------------|
  | 供应链 | 高 | 头部品牌自建原料基地 | 根据XXX报告，XX品牌拥有XX万亩茶园 |
  | 产品创新 | 高 | 周周上新 | 根据YYY数据，2024年头部品牌平均上新XX款 |
- 7.2 优先级排序

**八、机会与风险**
- 8.1 机会点：市场机会[列表+**数据支撑+来源**]、细分机会[列表+**数据支撑+来源**]
- 8.2 风险点：行业风险、竞争风险、政策/监管风险、产品/技术风险

**九、战略建议（选项式）**
- 9.1 选项A：[建议进入/建议深耕/持续观察]
  - 理由：[列表+**数据+来源**]
  - 建议条件：[列表]
- 9.2 选项B：[谨慎进入/选择性进入]
  - 理由：[列表+**数据+来源**]
  - 建议关注：[列表]
- 9.3 选项C：[不建议进入/暂缓]
  - 理由：[列表+**数据+来源**]

### 4.3 数据来源标注规范

**数据融合标注**（v3.0新增）：

| 标注格式 | 含义 | 示例 |
|---------|------|------|
| `【外部-欧睿】` | 仅单一外部源 | `2024年市场规模：480亿元【外部-欧睿】` |
| `【外部-欧睿 + 内部-销售数据】（已验证）` | 多源一致，差异<5% | `2024年市场规模：482.5亿元【外部-欧睿 + 内部-销售数据】（已验证，差异1%）` |
| `【外部-欧睿 vs 内部-销售数据】（差异15%，见冲突分析）` | 存在冲突，差异>10% | `2024年增速：15%【外部-欧睿 vs 内部-销售数据】（差异15%，见冲突分析）` |
| `【信息不足】` | 搜索后无结果 | `细分市场规模：信息不足（详见数据缺口说明）` |

**分级标注要求**：

**高可信（完整7字段）**：
```
来源类型：[报告/政府统计/上市公司财报]
具体来源：[完整标题]
发布机构/作者：[机构名]
发布时间：[YYYY-MM-DD]
数据时效：[如：2024年数据]
获取渠道：[如：XXX官网/数据库]
链接：[URL地址]
```

**中可信（简化标注）**：
```
来源：[报告标题]，[发布机构]，[发布时间]
链接：[URL或说明]
```

**低可信（基础标注+说明）**：
```
来源：[文章标题/网站名]，[日期]

```

**信息不足**：
```
来源：信息不足

```

## 五、分析规则与决策标准

### 5.1 数据融合规则

**引用**：`SOP_v3.0_融合与冲突处理规则.md`

**融合流程**：

```
并行获取L1/L2外部数据 + L3内部文件 → 智能比对 → 冲突检测 → 融合/冲突处理 → 输出结果
```

**冲突处理优先级**：

| 冲突类型 | 处理策略 | 优先级 |
|----------|----------|--------|
| 高可信 vs 中可信 | 采用高可信，标注差异 | P0 |
| 多源高可信冲突 | 标注差异，说明采用依据，请求用户决策 | P0 |
| 外部 vs 内部冲突 | 触发预警，提供A/B/C选项，不自动覆盖 | P0 |
| 时效性冲突 | 采用最新数据，标注旧数据时效 | P1 |
| 统计口径冲突 | 标注口径差异，分别列示 | P1 |

### 5.2 竞争强度评估标准

**五力强度量化指标参考**：

| 维度 | 高 | 中 | 低 |
|------|----|----|-----|
| **现有竞争** | CR5<40%，价格战频繁，产品高度同质化 | CR5 40-60%，价格战中等 | CR5>60%，市场集中，价格稳定 |
| **进入壁垒-资金** | <500万 | 500万-2000万 | >2000万 |
| **进入壁垒-供应链** | 无要求 | 部分自建 | 全链条自建 |
| **进入壁垒-品牌** | 无认知门槛 | 区域认知 | 全国性品牌认知 |
| **替代威胁** | 高替代、转换成本<5元 | 中等替代、5-15元 | 低替代、>15元 |

**关键成功因素优先级排序方法**：

```
评分维度：重要性(40%) + 稀缺性(30%) + 可获取性(30%)
排序规则：加权分高→低
```

### 5.3 生命周期判断标准

| 阶段 | 定性描述 | 量化参考指标 |
|------|----------|--------------|
| **导入期** | 新品类、增长快速但不稳定、企业数量增加、消费者认知低 | 渗透率<10%，品牌数量<10个，增长率>50% |
| **成长期** | 快速增长、市场规模扩大、竞争开始加剧、产品丰富度提升 | 渗透率10-30%，增长率15-50% |
| **成熟期** | 增长放缓、市场集中度提高、产品同质化、头部品牌固化 | 渗透率>30%，增长率5-15%，CR5>60% |
| **衰退期** | 负增长或零增长、消费转移、产能过剩、价格战 | 增长率<0%，连续2年负增长 |

**说明**：量化指标为参考标准，非绝对标准。若无法获取数据，可基于定性描述判断。

### 5.4 进入决策判断矩阵

| 维度 | 可进入 | 谨慎进入 | 不建议 |
|------|--------|----------|--------|
| **市场规模** | >100亿 | 50-100亿 | <50亿 |
| **增长率** | >15% | 5-15% | <5%或负增长 |
| **竞争强度** | 中低 | 中 | 高（CR5>60%） |
| **进入壁垒** | 可突破 | 较高 | 极高 |
| **功能差异化** | 有明显差异化空间 | 差异化不明显 | 已同质化 |
| **监管环境** | 明确/可申请 | 需进一步确认 | 限制性强 |

### 5.5 退出/调整决策

- 行业增长率持续为负超过2年
- 政策环境出现重大不利变化
- 竞争格局已形成稳定寡头（新进入者机会极小）
- 功能性宣称面临监管风险

## 九、接口定义（为全流程自动化准备）

### 9.1 输入（从上下文读取）
| 数据项 | 来源 | 说明 |
|--------|------|------|
| **全局参数** | `.sop/context/project-context.md` | 目标品类、地理范围、研究目的、功能方向（如有） |
| **前置模块输出** | 无 | 本模块为市场调研首个模块，无前置依赖 |

### 9.2 输出（写入上下文，供后续模块使用）
| 数据项 | 格式 | 接收模块 |
|--------|------|---------|
| **市场规模数据** | JSON/表格（历史 + 预测） | 1.2 竞争分析、2.1 市场定位 |
| **行业生命周期阶段** | 文本（阶段 + 依据） | 1.2 竞争分析、2.1 市场定位 |
| **关键成功因素 (KSFs)** | 清单（按优先级排序） | 1.2 竞争分析、4.1 产品组合规划 |
| **机会点与风险点** | 清单（带数据支撑） | 1.5 机会分析、2.1 市场定位 |
| **战略建议选项** | 选项列表（A/B/C） | 用户决策点（Checkpoint 1） |

### 9.3 阶段检查点（Checkpoint 1）
**执行完毕后，AI 必须执行**：
1. **输出摘要**：生成"Phase 1 调研摘要"（包含市场规模、生命周期、Top3 机会点）。
2. **请求确认**：
   - 若用户已有功能方向/品牌调性假设 → 验证其可行性并提供修正建议。
   - 若用户无明确方向 → 基于调研数据生成 3 个推荐方向（含市场规模、增速、竞争强度）。
3. **等待用户选择**：用户确认后，更新 `.sop/context/project-context.md`，再进入 1.2 竞争分析。

**注意**：在用户确认前，**不得自动进入下一模块**。
