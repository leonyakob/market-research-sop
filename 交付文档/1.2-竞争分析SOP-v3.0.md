# 1.2 竞争分析 SOP框架 v3.0

> **模板说明**：本文档为1.2竞争分析模块的v3.0版本，基于SOP v3.0通用模板构建。
> 核心升级：先格局后拆解（竞争格局战略分析 + 竞品深度拆解）、战略群组分析 + 竞品对标矩阵 + 感知图/定位图、双输出模式（人类可读报告 + 机器可读JSON）。

---

## 前置声明

### "求真务实"原则（5条规则）

AI在执行本SOP时，**必须严格遵循以下规则**：

1. **所有事实性信息必须来源于可验证的资料**（如学术文献、官方文档、权威出版物、可靠数据库、政府统计、上市公司财报）
2. **每一条关键结论应附带来源说明或推理依据**
3. **禁止生成无依据的推测、臆断或虚构内容**
4. **当缺乏足够信息支持时，应明确说明"信息不足"，而非补全或猜测**
5. **如存在多种观点，应说明不同来源及其差异，而非合并或简化为单一结论**

### 本模块在品牌策划全案中的位置

```
市场调研
├── 1.1 行业研究 ← 已完成
├── 1.2 竞争分析 ← [本模块]
├── 1.3 消费者研究
├── 1.4 趋势分析
└── 1.5 机会分析
```

### 与其他模块的依赖关系

| 依赖模块 | 关系类型 | 说明 |
|----------|----------|------|
| 1.1 行业研究 | 强依赖 | 需要1.1输出的市场规模数据、行业生命周期阶段、关键成功因素KSFs、机会点与风险点 |
| 1.3 消费者研究 | 并行 | 可并行执行，消费者画像数据可补充竞品用户分析 |
| 1.5 机会分析 | 输出依赖 | 本模块输出的竞争机会清单作为1.5的输入 |

### 本模块结构概览

本报告包含4个核心模块（竞争格局总览、竞品深度拆解、竞品对标矩阵、竞争机会识别）+ 4张数据集表格。

分析维度：9个（6必须 + 3可选）。必须维度：市场地位、产品策略、定价策略、渠道布局、品牌与营销、竞争动态与趋势。可选维度：用户口碑、供应链能力、财务与资本（每个有量化触发条件）。

需要数据：1.1输出的行业数据、竞品市场份额、产品矩阵、价格体系、渠道覆盖、品牌定位、营销动作、用户评价、供应链信息、融资与财务数据。

## 一、触发条件

### 1.1 何时执行本模块

本模块在以下场景触发执行：

- **新品牌进入决策**：完成1.1行业研究后，需明确"跟谁打、怎么打"
- **竞品对标分析**：已有明确竞品名单，需系统性拆解对手策略
- **竞争格局变化**：行业出现重大变化（新玩家进入、价格战、政策变动），需重新评估竞争态势
- **战略调整验证**：现有竞争策略执行一段时间后，需评估效果并调整

### 1.2 前置模块完成状态

| 前置模块 | 完成状态要求 | 验证方式 |
|----------|--------------|----------|
| 1.1 行业研究 | 必须完成 | 检查输出文档完整性：市场规模数据、行业生命周期阶段、KSFs清单、机会与风险点 |
| 1.3 消费者研究 | 建议完成 | 可选，消费者画像数据可补充竞品用户分析维度 |

**前置数据验证清单**：
- [ ] 1.1输出文件存在且包含市场规模数据（历史+预测）
- [ ] 1.1输出文件包含行业生命周期阶段判断
- [ ] 1.1输出文件包含KSFs清单（至少3个，带优先级排序）
- [ ] 1.1输出文件包含机会点与风险点（带数据支撑）

## 二、输入参数

### 2.1 用户需提供（必填）

#### 核心参数

| 参数 | 说明 | 示例 |
|------|------|------|
| **目标品类** | 研究的具体产品品类（需与1.1一致） | 无糖功能性茶饮 |
| **指定竞品** | 用户已知需重点分析的竞品品牌（可选，AI会自动补充Top 5） | 东方树叶、三得利、元气森林 |

#### 可选文件输入

| 文件类型 | 说明 | 格式要求 |
|----------|------|----------|
| 竞品价格监控表 | 已有竞品价格追踪数据 | .xlsx, .csv |
| 竞品营销案例 | 内部收集的竞品营销活动资料 | .pdf, .docx, .pptx |
| 竞品产品清单 | 竞品SKU及规格信息 | .xlsx, .csv |

**内部文件上传说明**：
- 文件存放路径：`.sop/internal/1.2_竞争分析/`
- 命名规范：`YYYYMMDD_类型_描述.ext`（如：`20260405_价格监控_东方树叶.xlsx`）
- AI处理规则：参见 `.sop/config/internal-files.yaml`
- 标注规范：`【内部-类型-文件名】`（不公开路径，涉密）

### 2.2 AI需确认（含默认值机制）

| 参数 | AI询问方式 | 选项示例 | 默认值（10分钟内未回复） |
|------|------------|----------|------------------------|
| **地理范围** | "请选择竞争分析的市场区域：" | 中国市场 / 全球市场 / 区域市场（如华东、华南） | 中国市场 |
| **分析深度** | "请选择竞品分析的深度：" | 深度模式（每个维度独立成段，有数据+分析+洞察+启示）/ 标准模式（概要分析） | 深度模式 |
| **时间范围** | "请选择竞争动态监测的时间范围：" | 近1年 / 近2年 / 近3年 | 近1年 |

**执行规则（强制交互）**：

1. **必须暂停**：AI 输出参数确认问题后，**必须立即停止生成**，不得继续执行搜索或分析。
2. **等待回复**：等待用户明确回复（选择选项、输入自定义值或回复"使用默认值"）。
3. **禁止自作主张**：严禁在未获用户确认前使用默认值。
4. **默认值触发**：仅当用户明确回复"使用默认值"或"跳过"时，方可使用默认值。
5. **报告中标注确认情况**：如"本研究参数已确认：地理范围=中国市场，分析深度=深度模式"。

### 2.3 竞品选择规则

**AI自动选择（Top 5）**：
- 基于1.1行业研究输出的市场份额数据，自动选取市场份额排名前5的品牌
- 若1.1未提供完整份额数据，则基于公开可获取的市场排名信息选取
- 入选依据必须标注：市场份额数值、数据来源、数据年份

**用户手动添加**：
- 用户可在"指定竞品"参数中补充AI未选中的品牌
- 用户指定竞品与AI自动选择的竞品合并为最终竞品集
- 去重后，竞品总数建议控制在5-8个，超过8个时标注"竞品数量较多，建议聚焦核心竞品"

**竞品入选标准**：
- 必须条件：在目标品类中有实际产品销售
- 优先条件：市场份额前5、或用户指定、或近期有重大竞争动作
- 排除条件：已退出市场、或仅试水未正式运营的品牌

## 三、数据调用策略

> **数据源配置**：`.sop/config/data-acquisition.yaml`
> 
> **当前状态**：Exa/Tavily 外部搜索（网页内容 → AI提取数据）
> 
> **未来状态**：自建数据源 API（自然语言查询 → 结构化JSON → AI直接使用）
> 
> **切换方式**：修改配置文件中的 `primary` 字段，**SOP无需任何改动**

### 3.1 数据调用指令

**配置引用**：`.sop/config/data-acquisition.yaml`

**执行逻辑**：

```
AI生成自然语言查询 → 调用主数据源（当前Exa，未来internal_api）
  → 若失败 → 自动降级fallback（Tavily）
  → 返回数据 → AI提取洞察、填入报告
```

**数据调用指令模板**（AI按此格式生成查询）：

| 数据维度 | 自然语言查询模板 | 预期返回内容 |
|---------|----------------|-------------|
| **市场份额** | `获取{品类}在{地理范围}的主要品牌市场份额排名、CR5/CR10数据、品牌梯队划分，包含各品牌具体份额百分比` | 品牌份额排名 + CR5 + 梯队分析 + 具体数值 |
| **产品矩阵** | `获取{竞品名}在{品类}的产品线布局，包含SKU数量、产品规格、核心卖点、上市时间` | 产品清单 + 规格参数 + 卖点 + 时间线 |
| **定价体系** | `获取{竞品名}在{品类}的价格体系，包含各SKU定价、促销策略、价格带分布` | 价格表 + 促销信息 + 价格带分析 |
| **渠道覆盖** | `获取{竞品名}在{地理范围}的渠道布局，包含线上渠道（天猫/京东/抖音等）和线下渠道（商超/便利店/餐饮等）的覆盖情况` | 渠道清单 + 覆盖范围 + 渠道策略 |
| **品牌定位** | `获取{竞品名}的品牌定位、目标人群、品牌主张、视觉风格、营销调性` | 品牌定位描述 + 目标人群 + 营销风格 |
| **营销动作** | `获取{竞品名}在{时间范围}内的主要营销活动、广告投放、联名合作、社交媒体动作` | 营销事件清单 + 时间线 + 效果数据 |
| **用户口碑** | `获取{竞品名}在主流电商平台和社交媒体的用户评价、评分、好评点、差评点` | 评分数据 + 评价样本 + 正负面关键词 |
| **供应链信息** | `获取{竞品名}的供应链布局、生产基地、原料来源、物流体系、产能信息` | 供应链结构 + 产能数据 + 关键节点 |
| **融资与财务** | `获取{竞品名}的融资历程、估值变化、营收数据、盈利能力、投资方信息` | 融资时间线 + 财务指标 + 投资方 |
| **竞争动态** | `获取{品类}在{地理范围}{时间范围}内的重大竞争事件，包含新品发布、价格调整、渠道扩张、品牌升级` | 竞争事件时间线 + 影响评估 |

**占位符替换规则**：
- `{品类}` → 用户输入的目标品类（如"无糖功能性茶饮"）
- `{地理范围}` → 确认的地理范围（如"中国"）
- `{竞品名}` → 具体竞品品牌名称
- `{时间范围}` → 确认的时间范围（如"近1年"）

### 3.2 当前阶段：Exa/Tavily 搜索

**当前主数据源**：Exa（`primary: exa`）
**降级数据源**：Tavily（`fallback: tavily`）

**返回格式（当前）**：

```json
{
  "results": [
    {
      "title": "页面标题",
      "url": "https://...",
      "publishedDate": "2024-01-15",
      "text": "网页完整文本内容...",
      "highlights": ["AI提取的关键片段1", "关键片段2"],
      "summary": "AI生成的页面摘要"
    }
  ]
}
```

**AI处理方式**：
1. 从返回的网页文本/highlights/summary中**提取**关键数据（数值、单位、年份、来源）
2. 交叉验证多个结果的一致性
3. 按来源标注规范记录数据来源
4. 将提取的数据填入报告

### 3.3 未来阶段：自建数据源 API

**未来主数据源**：自建数据源（`primary: internal_api`）

**返回格式（预期）**：

```json
{
  "query": "原始查询",
  "answer": "综合回答（整合多源后的分析性文本，包含趋势说明、口径说明、竞争格局背景）",
  "metrics": [
    {"name": "指标名称", "value": 数值, "unit": "单位", "year": 年份, "confidence": "高/中/低"}
  ],
  "sources": [
    {"name": "来源名称", "type": "来源类型", "url": "链接"}
  ],
  "notes": "补充说明（口径差异、数据冲突处理说明等）"
}
```

**AI处理方式**：
1. `metrics` → 直接填入数据表
2. `answer` → 理解背景，提取洞察写入分析段落
3. `notes` → 在报告中标注口径说明
4. `sources` → 生成引用标注

**切换步骤**（技术人员执行）：
1. 部署自建数据源API
2. 配置 `.sop/config/data-acquisition.yaml` 中的 `endpoint` 和 `auth_token`
3. 修改 `primary: internal_api`
4. 测试验证
5. **SOP文件无需任何修改**

### 3.4 内部文件处理

**配置文件引用**：`.sop/config/internal-files.yaml`

**文件读取规则**：

| 文件大类 | 读取方式 | 标注规范 | 隐私级别 |
|----------|----------|----------|---------|
| 竞品与情报类 | 来源标注→时效性检查→交叉验证 | 【内部-竞品情报-文件名】 | 中/高 |
| 市场与销售类 | 数据清洗→字段标准化→样本校验 | 【内部-市场数据-文件名】 | 中/高 |
| 产品与供应链类 | 成本拆分→工艺标准化→风险标注 | 【内部-供应链-文件名】 | 高 |
| 财务与战略类 | 高度脱敏→比例保留→假设标注 | 【内部-财务-文件名】 | 高 |

**内部文件标注规范**：

```
【内部-大类-子类_{标识}_{日期}.{格式}】
- 示例：【内部-竞品情报-价格监控_东方树叶_202412.xlsx】
- 示例：【内部-竞品情报-营销复盘_元气森林_联名活动_202411.pptx】
```

**融合规则**：内部数据与外部数据并行比对，差异<5%标注"已验证"，差异>10%触发冲突预警。详见 `SOP_v3.0_融合与冲突处理规则.md`。

### 3.5 AI自主补充查询

**补充触发条件表**：

| 条件 | 判断方法 | 阈值 | 补充查询指令 |
|------|----------|------|-------------|
| 竞品数据缺口 | 某竞品在核心维度无数据 | ≥1个核心维度无数据 | 补充查询：竞品名 + 维度关键词 |
| 数据冲突 | 多源数据差异 > 10% | >10% | 补充查询：第三方验证数据 + 统计口径说明 |
| 时效性不足 | 最新数据超过1年 | >1年 | 补充查询：竞品名 + 最新动态 + 2025/2026 |
| 可选维度触发 | 满足量化触发条件 | 见5.5节 | 按触发条件执行对应维度查询 |

**数据调用失败处理流程**：

```
主数据源调用失败 → 自动降级fallback（Tavily）
  → 同义词扩展查询（如"市场份额"→"市场占有率"→"市场排名"）
  → 扩大查询范围（如具体竞品名→品类+竞品类型）
  → 若仍无结果 → 标注"信息不足"并说明已尝试路径
  → 继续执行其他维度，不阻塞流程
```

## 四、输出格式

### 4.1 报告结构（4个模块）

**一、竞争格局总览**
- 1.1 市场集中度分析：CR5/CR10数值、品牌梯队划分（头部/腰部/尾部）、集中度趋势
- 1.2 战略群组分析：基于关键维度（价格带×渠道覆盖×产品定位）划分战略群组，输出战略群组表
- 1.3 感知图/定位图：选取2个核心维度（如价格×功能、或传统×创新）绘制竞品感知图，输出定位图数据
- 1.4 竞争强度评估：基于1.1波特五力结论（直接引用），补充本模块竞争强度量化评分

**二、竞品深度拆解（每个竞品独立成段）**
- 2.1 竞品档案：品牌背景、市场份额、核心产品、目标人群
- 2.2 产品策略：产品线宽度与深度、核心SKU、差异化卖点、上新频率、产品生命周期管理
- 2.3 定价策略：价格带分布、定价逻辑（成本导向/价值导向/竞争导向）、促销策略、价格弹性表现
- 2.4 渠道布局：线上渠道（平台分布、自营/代运营）、线下渠道（渠道类型、覆盖区域、终端数量）、渠道策略特点
- 2.5 品牌与营销：品牌定位、品牌主张、视觉风格、营销调性、核心营销动作（近{时间范围}）
- 2.6 竞争动态与趋势：近{时间范围}内重大动作（新品/调价/渠道/营销/资本）、战略意图推断、未来趋势预判
- 2.7 可选维度（触发时展开）：
  - 用户口碑：评分、评价样本、好评点、差评点、口碑趋势
  - 供应链能力：生产基地、原料来源、产能、供应链优势/风险
  - 财务与资本：融资历程、估值、营收、盈利能力、投资方背景

**三、竞品对标矩阵**
- 3.1 多维度对标表：所有竞品在9个维度上的横向对比（表格形式）
- 3.2 关键成功因素覆盖度：基于1.1输出的KSFs，评估各竞品的覆盖情况
- 3.3 竞争优劣势总结：各竞品的核心优势与明显短板

**四、竞争机会识别**
- 4.1 市场空白点：未被满足的需求、未被覆盖的价格带/渠道/人群
- 4.2 竞品薄弱环节：竞品普遍存在的短板、用户集中投诉的问题
- 4.3 差异化机会：可建立竞争壁垒的差异化方向
- 4.4 竞争策略建议（决策检查点）：基于以上分析，输出策略建议选项（标注为报告内决策检查点，非正式输出接口）

### 4.2 各维度分析深度定义

**深度模式**（默认）：
每个分析维度独立成段，包含以下4层结构：
1. **数据**：具体数值、事实、来源标注
2. **分析**：数据背后的逻辑和趋势解读
3. **洞察**：对我方品牌的启示和可借鉴点
4. **我方启示**：可直接转化为行动的建议

**标准模式**：
每个分析维度为概要描述，包含核心数据和简要分析，不展开洞察和启示。

### 4.3 数据集格式（4张表）

## 五、分析规则与决策标准

### 5.1 数据融合规则

**引用**：`SOP_v3.0_融合与冲突处理规则.md`

**融合流程**：

```
并行获取L1/L2外部数据 + L3内部文件 → 智能比对 → 冲突检测 → 融合/冲突处理 → 输出结果
```

**冲突处理优先级**：

| 冲突类型 | 处理策略 | 优先级 |
|----------|----------|--------|
| 高可信 vs 中可信 | 采用高可信，标注差异 | P0 |
| 多源高可信冲突 | 标注差异，说明采用依据，请求用户决策 | P0 |
| 外部 vs 内部冲突 | 触发预警，提供A/B/C选项，不自动覆盖 | P0 |
| 时效性冲突 | 采用最新数据，标注旧数据时效 | P1 |
| 统计口径冲突 | 标注口径差异，分别列示 | P1 |

### 5.2 上游数据引用规则（1.1 → 1.2）

**核心原则**：SOP 模块间是"信任链"关系，不是"审计链"关系。每个模块信任上游输出，但保持可追溯和冲突检测能力。

**执行规则**：

| 规则 | 具体做法 | 示例 |
|------|---------|------|
| **直接引用** | 1.1 输出的数据（市场规模、生命周期、KSFs、机会风险）可直接使用，不重新验证 | 使用 1.1 输出的"市场规模 480 亿元" |
| **标注来源** | 使用时标注 `【1.1 行业研究输出】` | `市场规模：480亿元【1.1 行业研究输出】` |
| **冲突检测** | 若 1.2 自行获取的数据与 1.1 输出差异>10%，触发冲突预警 | `【冲突预警】1.1输出480亿 vs 外部搜索520亿，差异8.3%，已验证` |
| **置信度继承** | 1.1 标注为"低可信"的数据，1.2 使用时需标注"继承低可信" | `细分市场规模：XX亿元【1.1 行业研究输出，继承低可信】` |
| **不重新验证** | 1.2 不重复 1.1 的数据获取工作 | 不重新搜索市场规模 |

**冲突处理流程**：
```
1.2 自行获取数据 → 与 1.1 输出比对
  → 差异<5%：标注"已验证"
  → 差异 5-10%：标注差异值，采用 1.1 数据（上游模块优先）
  → 差异>10%：触发冲突预警，在报告中标注双方数据及差异，请求用户决策
```

### 5.3 战略群组分析规则

**分组维度**（选取2-3个核心维度）：
- 价格带：高端（>XX元）、中端（XX-XX元）、大众（<XX元）
- 渠道覆盖：全渠道、线上为主、线下为主
- 产品定位：功能导向、性价比导向、创新导向、品牌导向

**分组规则**：
1. 基于1.1行业研究输出的KSFs，选取与KSFs最相关的2-3个维度
2. 每个维度划分为2-3个等级
3. 将各竞品映射到维度组合中
4. 同一组合内的品牌归为同一战略群组
5. 群组数量建议3-5个，过多则合并相似群组

**输出要求**：
- 战略群组表（表3）
- 各群组的核心竞争策略描述
- 各群组之间的竞争关系分析
- 我方品牌可进入的群组位置建议

### 5.4 感知图/定位图规则

**维度选择**：
- 选取2个对消费者决策影响最大的维度作为X轴和Y轴
- 常见维度组合：价格×功能、传统×创新、大众×高端、实用×情感
- 维度选择依据：1.1消费者研究数据（如有）或行业KSFs

**绘制规则**：
1. X轴和Y轴各划分为3-5个刻度
2. 将各竞品映射到二维坐标系中
3. 标注各竞品的品牌名称
4. 标注各象限的特征描述
5. 标注市场空白区域（机会点）

**输出要求**：
- 感知图数据（JSON格式，包含各竞品的X/Y坐标值）
- 图表元数据（轴标签、刻度、象限描述）
- 空白区域标注（潜在机会点）

### 5.5 可选维度触发条件（量化，非模糊描述）

**用户口碑**：
- **触发条件**：该竞品在主流平台（天猫/京东/小红书）有≥50条可获取评价
- **执行动作**：采集评价样本，计算评分分布，提取高频好评/差评关键词
- **跳过标注**：`【信息不足】该维度数据不可获取（原因：该竞品在主流平台可获取评价不足50条）`

**供应链能力**：
- **触发条件**：该竞品有公开供应链信息（招股书/财报/深度报道≥2篇）
- **执行动作**：提取生产基地、原料来源、产能、供应链优势/风险信息
- **跳过标注**：`【信息不足】该维度数据不可获取（原因：该竞品无公开供应链信息，招股书/财报/深度报道不足2篇）`

**财务与资本**：
- **触发条件**：该竞品有公开融资信息或营收数据（非上市公司且无公开融资记录则跳过）
- **执行动作**：提取融资历程、估值变化、营收数据、盈利能力、投资方信息
- **跳过标注**：`【信息不足】该维度数据不可获取（原因：该竞品为非上市公司且无公开融资记录）`

### 5.6 竞争强度评估

**竞争强度量化评分**（1-5分）：

| 评分维度 | 权重 | 1分（极低） | 3分（中等） | 5分（极高） |
|----------|------|-------------|-------------|-------------|
| 市场集中度 | 25% | CR5>80%，寡头垄断 | CR5 50-80%，中度集中 | CR5<50%，高度分散 |
| 价格竞争烈度 | 20% | 价格稳定，无价格战 | 偶发促销，价格波动小 | 频繁价格战，利润压缩 |
| 产品差异化程度 | 20% | 高度差异化，各有壁垒 | 部分差异化，可替代 | 高度同质化，难以区分 |
| 新进入者威胁 | 15% | 壁垒极高，新进入者极少 | 壁垒中等，偶有新玩家 | 壁垒低，新品牌频出 |
| 替代品威胁 | 10% | 无直接替代品 | 有替代品但转换成本高 | 替代品多且转换成本低 |
| 渠道争夺烈度 | 10% | 渠道充足，无争夺 | 部分渠道竞争激烈 | 核心渠道被头部垄断 |

**竞争强度等级**：
- 1.0-2.0分：低强度竞争（市场格局稳定，新进入者有机会）
- 2.1-3.5分：中等强度竞争（需差异化策略突破）
- 3.6-5.0分：高强度竞争（需强资源投入或创新模式）

### 5.7 竞争策略建议判断矩阵（决策检查点）

> **注意**：本矩阵仅作为报告内决策检查点使用，**不作为正式输出接口**。

| 竞争态势 | 推荐策略 | 适用条件 | 风险提示 |
|----------|----------|----------|----------|
| 市场空白明显 | 差异化切入 | 有明确未被满足的需求 | 需验证需求真实性 |
| 竞品普遍薄弱 | 优势攻击 | 我方在该维度有优势 | 竞品可能快速补齐 |
| 头部垄断稳固 | 侧翼突围 | 找到头部未覆盖的细分市场 | 需控制投入规模 |
| 新锐品牌活跃 | 跟随+微创新 | 学习新锐打法并优化 | 避免同质化竞争 |
| 价格战激烈 | 价值升级 | 有产品力支撑溢价 | 需教育市场接受高价 |

## 九、接口定义（为全流程自动化准备）

### 9.1 输入

| 数据项 | 来源 | 说明 |
|--------|------|------|
| **全局参数** | `.sop/context/project-context.md` | 目标品类、地理范围、分析深度、时间范围 |
| **1.1 行业研究输出** | `.sop/context/1.1-output.json` | 市场规模数据、行业生命周期阶段、KSFs清单、机会点与风险点 |
| **用户指定竞品** | 用户输入 | 用户手动添加的竞品品牌列表 |

### 9.2 输出（10项正式输出）

| 序号 | 输出项 | 格式 | 接收模块/用途 |
|------|--------|------|--------------|
| 1 | **竞品集与入选依据** | JSON（品牌列表 + 入选原因 + 市场份额 + 数据来源） | 1.5机会分析、2.1市场定位、4.1产品组合规划 |
| 2 | **战略群组结果** | JSON + 表格（群组划分 + 包含品牌 + 核心策略） | 1.5机会分析、2.1市场定位 |
| 3 | **定位图数据** | JSON + 图表元数据（竞品坐标 + 轴标签 + 象限描述 + 空白区域） | 2.1市场定位、1.5机会分析 |
| 4 | **竞品对标矩阵** | 表格/JSON（多竞品×多维度横向对比） | 1.5机会分析、2.1市场定位、4.1产品组合规划 |
| 5 | **竞品产品策略画像** | JSON list（每个竞品的产品线宽度、核心SKU、差异化卖点、上新频率） | 4.1产品组合规划 |
| 6 | **竞品定价策略画像** | JSON list + 表格（每个竞品的价格带、定价逻辑、促销策略、价格弹性） | 4.1产品组合规划、2.1市场定位 |
| 7 | **竞品渠道画像** | JSON list（每个竞品的线上/线下渠道分布、覆盖区域、渠道策略特点） | 渠道策略模块 |
| 8 | **竞品品牌定位画像** | JSON list（每个竞品的品牌定位、目标人群、品牌主张、营销调性） | 2.1市场定位 |
| 9 | **KSF覆盖矩阵** | 表格/JSON（各竞品对1.1输出KSFs的覆盖度评估） | 1.5机会分析、2.1市场定位 |
| 10 | **竞争机会清单** | 表格/JSON（市场空白点 + 竞品薄弱环节 + 差异化机会 + 优先级排序） | 1.5机会分析、2.1市场定位、用户决策点 |

### 9.3 条件输出（4项，满足触发条件时输出）

| 序号 | 输出项 | 触发条件 | 格式 | 接收模块/用途 |
|------|--------|----------|------|--------------|
| C1 | **用户口碑洞察** | 该竞品在主流平台（天猫/京东/小红书）有≥50条可获取评价 | JSON（评分分布 + 高频关键词 + 好评点 + 差评点 + 评价样本） | 1.3消费者研究、2.1市场定位 |
| C2 | **供应链能力信号** | 该竞品有公开供应链信息（招股书/财报/深度报道≥2篇） | JSON（生产基地 + 原料来源 + 产能 + 优势/风险） | 4.1产品组合规划、供应链策略 |
| C3 | **财务与资本信号** | 该竞品有公开融资信息或营收数据（非上市公司且无公开融资记录则跳过） | JSON（融资历程 + 估值 + 营收 + 盈利能力 + 投资方） | 1.5机会分析、投资决策 |
| C4 | **竞品动态监测点** | 近6-12个月有显著动作（新品/调价/渠道/营销/资本） | JSON（事件时间线 + 影响评估 + 趋势预判） | 1.4趋势分析、1.5机会分析 |

### 9.4 阶段检查点（Checkpoint 2）

**执行完毕后，AI 必须执行**：

1. **输出摘要**：生成"Phase 1.2 竞争分析摘要"（包含竞品集、战略群组结论、Top 3竞争机会）。
2. **请求确认**：
   - 展示竞品对标矩阵核心结论
   - 展示竞争机会清单
   - 展示竞争策略建议选项（决策检查点）
3. **等待用户选择**：用户确认后，更新 `.sop/context/project-context.md`，再进入 1.3 消费者研究。

**注意**：在用户确认前，**不得自动进入下一模块**。

### 9.5 输出数据格式规范（JSON Schema 参考）

**竞品集与入选依据**：
```json
{
  "competitors": [
    {
      "name": "品牌名称",
      "selection_reason": "AI Top 5 / 用户指定 / 重大竞争动作",
      "market_share": "XX%",
      "market_share_source": "数据来源",
      "market_share_year": "年份"
    }
  ],
  "total_competitors": 5,
  "generated_at": "YYYY-MM-DD"
}
```

**竞争机会清单**：
```json
{
  "opportunities": [
    {
      "type": "市场空白 / 竞品薄弱 / 差异化机会",
      "description": "机会描述",
      "evidence": "数据支撑",
      "priority": "高/中/低",
      "actionable": "可执行建议"
    }
  ],
  "total_opportunities": 3,
  "generated_at": "YYYY-MM-DD"
}
```

**模块间数据存储**：
- 存储路径：`.sop/context/1.2-output.json`
- 存储格式：JSON
- 更新时机：本模块执行完毕且用户确认后
