# 1.3 消费者研究 SOP框架 v3.0

> **模板说明**：本文档为1.3消费者研究模块的v3.0版本，基于SOP v3.0通用模板构建。  
> 核心方法：决策驱动主轴选择（二元检查清单 + 加权计算）+ 强制覆盖层 + 证据门控 + 上游数据融合 + 下游结构化交接。

---

## 前置声明

### "求真务实"原则（5条规则）

AI在执行本SOP时，**必须严格遵循以下规则**：

1. **所有事实性信息必须来源于可验证的资料**（如学术文献、官方文档、权威出版物、可靠数据库、政府统计、上市公司财报）
2. **每一条关键结论应附带来源说明或推理依据**
3. **禁止生成无依据的推测、臆断或虚构内容**
4. **当缺乏足够信息支持时，应明确说明"信息不足"，而非补全或猜测**
5. **如存在多种观点，应说明不同来源及其差异，而非合并或简化为单一结论**

### 本模块在品牌策划全案中的位置

```
市场调研
├── 1.1 行业研究 ← 已完成
├── 1.2 竞争分析 ← 已完成
├── 1.3 消费者研究 ← [本模块]
├── 1.4 趋势分析
└── 1.5 机会分析
```

### 与其他模块的依赖关系

| 依赖模块 | 关系类型 | 说明 |
|----------|----------|------|
| 1.1 行业研究 | 强依赖 | 需要1.1输出的社会文化信号、KSFs、行业阶段、机会与风险框架 |
| 1.2 竞争分析 | 强依赖（条件增强） | 需要1.2输出的竞品机会ID、口碑可用状态、未满足需求与切换触发 |
| 1.4 趋势分析 | 输出依赖 | 本模块输出人群张力、行为变化信号，供趋势追踪 |
| 1.5 机会分析 | 输出依赖 | 本模块输出高增长/高痛点低匹配人群，供机会优先级排序 |
| 2.1 市场定位 | 输出依赖 | 本模块输出目标人群优先级、价值驱动、定位张力与证明点需求 |

## 一、触发条件

### 1.1 何时执行本模块

本模块在以下场景触发执行：

- **新品牌进入**：已完成1.1/1.2，需明确"优先服务谁"与"为什么是他们"。
- **产品线扩展**：现有品类下新增子线，需识别可承接的新客群或边缘人群。
- **品牌重塑**：品牌主张或人群心智偏移，需重建核心目标消费者定义。
- **营销策略调整**：获客成本、转化率或复购表现异常，需定位决策链路阻塞点。

### 1.2 前置模块完成状态

| 前置模块 | 完成状态要求 | 验证方式 |
|----------|--------------|----------|
| 1.1 行业研究 | 必须完成 | 检查是否存在行业规模/阶段/KSFs/机会风险输出 |
| 1.2 竞争分析 | 必须完成（可降级） | 检查是否存在竞品机会清单、口碑数据可用状态、竞争覆盖信息 |

**前置验证清单**：
- [ ] 已读取1.1输出，并可提取 social_cultural_signals / ksfs / industry_lifecycle / opportunities_risks
- [ ] 已读取1.2输出，并可提取 sentiment_available / top_unmet_needs / top_switching_triggers / positioning_gaps / competitive_whitespace / opportunity_ids
- [ ] 若1.2口碑数据不可用，已在本模块标记条件降级，不阻塞主流程
- [ ] 若1.1整体缺失，已判定前置条件失败并停止正式执行
- [ ] 若1.2整体缺失，仅在用户明确同意时进入有限探索模式，并标注"不用于下游决策"

### 2.1 用户需提供（必填）

| 参数 | 说明 | 示例 |
|------|------|------|

| **研究目的** | 消费者研究业务目标（4选1） | 新品牌进入 |

**研究目的标准选项（4选1）**：
1. 新品牌进入
2. 产品线扩展
3. 品牌重塑
4. 营销策略调整

### 2.2 AI需确认（含默认值机制）

| 参数 | AI询问方式 | 选项示例 | 默认值（仅在用户明确同意时） |
|------|------------|----------|------------------------------|
| **地理范围** | "请选择消费者研究范围：" | 中国市场 / 全球市场 / 区域市场 | 中国市场 |
| **分析深度** | "请选择人群细分深度：" | 精简（2-3类）/ 标准（3-5类）/ 深度（5类+含更多扩展证据） | 标准（3-5类） |

### 2.3 可选文件输入

| 文件类型 | 说明 | 格式要求 |
|----------|------|----------|
| 调研数据 | 问卷、抽样、人群统计数据 | .xlsx, .csv |
| 访谈纪要 | 定性访谈、焦点小组、用户语录 | .docx, .pdf |
| 销售数据 | 复购、客单价、渠道转化 | .xlsx, .csv |

**内部文件处理说明**：
- 数据获取配置引用：`.sop/config/data-acquisition.yaml`
- 内部文件配置引用：`.sop/config/internal-files.yaml`
- 内部文件标注规则：仅写文件名，不写路径（如 `【内部-市场数据-消费者洞察_功能饮料_20260407.xlsx】`）

### 2.4 强制交互规则（关键）

1. **必须暂停**：AI 输出参数确认问题后，**必须立即停止生成**，不得继续执行搜索或分析。  
2. **等待回复**：等待用户明确回复（选择选项、输入自定义值或回复"使用默认值"）。  
3. **禁止自作主张**：严禁在未获用户确认前使用默认值。  
4. **默认值触发**：仅当用户明确回复"使用默认值"或"跳过"时，方可使用默认值。  
5. **报告中标注确认情况**：如"本研究参数已确认：地理范围=中国市场，分析深度=标准（3-5类）"。

## 三、数据调用策略

> **数据源配置**：`.sop/config/data-acquisition.yaml`  
> **内部文件配置**：`.sop/config/internal-files.yaml`

### 3.1 外部数据调用执行逻辑

```
AI生成自然语言查询 → 调用主数据源（当前 primary: exa）
  → 若失败自动降级 fallback: tavily
  → 返回网页证据/摘要 → AI提取结构化证据
  → 填入人群卡片与JSON输出
```

### 3.2 10条自然语言查询模板（附录A一致）

| 2 | **购买驱动与需求优先级** | `获取{地理范围}{品类}消费者的核心购买驱动及需求优先级，优先区分功能价值、情感价值、社交价值、风险规避、便利性，并返回各驱动的占比、排序或重要性` |
| 3 | **使用场景与任务情境** | `获取{地理范围}{品类}的主要使用场景与任务情境，包含发生时间、地点、伴随活动、使用目标、社交情境，并返回场景频次、重要性或典型人群差异` |
| 4 | **购买决策路径与关键触点** | `获取{地理范围}{品类}消费者从认知到购买的决策路径，包含信息来源、关键评估标准、比较对象、购买触发因素、放弃障碍、决策周期及关键节点` |
| 5 | **价格接受度与促销反应** | `获取{地理范围}{品类}消费者的价格接受度与促销反应，优先返回可接受价格带、价格敏感度分层、促销响应方式、溢价接受条件及对应人群差异` |
| 6 | **购买渠道与转化偏好** | `获取{地理范围}{品类}消费者的购买渠道与转化偏好，包含线上平台、线下门店、社交电商或其他主要渠道的使用占比、偏好排序、选择原因及典型适用场景` |
| 7 | **品牌关系与切换行为** | `获取{地理范围}{品类}消费者的品牌关系与切换行为，包含常购/主用品牌、兼用品牌、复购倾向、跨品牌切换频率、最近一次切换原因及忠诚驱动因素` |

| 9 | **痛点、未满足需求与使用障碍** | `获取{地理范围}{品类}消费者在选择、购买、使用或复购过程中的主要痛点、未满足需求、抱怨点与使用障碍，并返回出现频率、影响程度及典型人群差异` |
| 10 | **购买/使用频率与品类涉入度** | `获取{地理范围}{品类}消费者的购买频次、使用频率、使用强度、决策涉入度与决策复杂度，优先返回高频/中频/低频用户占比、典型周期及不同频次人群的行为差异` |

### 3.3 内部文件处理规则

| 规则项 | 处理要求 |
|--------|----------|
| 文件分类 | 按 `.sop/config/internal-files.yaml` 的 category/type 归类处理 |
| 隐私处理 | 高隐私先脱敏再引用；标注不暴露路径 |
| 引用格式 | `【内部-大类-子类_标识_日期.ext】` |
| 融合方式 | 内外部并行比对，差异明显时触发冲突处理 |

### 3.4 AI自主补充查询触发条件

| 条件 | 判断方法 | 阈值 | 补充动作 |
|------|----------|------|----------|
| 数据缺口 | 必须维度缺失 | ≥1个必须维度无可用证据 | 补查同义查询与替代来源 |
| 数据冲突 | 多源关键指标差异 | >10% | 启动冲突协议并保留双值 |
| 时效不足 | 最新证据过旧 | >12个月 | 补查近12个月来源 |
| 人群不可分 | 主轴打分接近或证据不足 | 分差<1或置信度低 | 触发混合主轴/降级模式 |

## 四、输出格式

### 4.1 人群画像报告结构

输出为**3-5类人群（标准模式）**，每类人群必须按四层结构给出：

1. **定义层（7项）**：ID、名称、主轴定位、判定条件、排除条件、规模、增速  
2. **核心层（8项）**：人口统计、购买动机、使用场景、渠道偏好、价格敏感度、品牌忠诚度、竞品使用、未满足需求  
3. **扩展层（5项）**：价值观/生活方式、关键决策因素、购买障碍、媒体接触习惯、购后满意度/NPS  
4. **决策层（7项）**：定位/产品/定价/渠道/传播启示、优先级+依据、竞争覆盖度

### 4.2 数据集格式

| 数据集 | 必含字段 | 用途 |
|--------|----------|------|
| 人群清单表 | segment_id, segment_name, primary_axis_position, priority | 汇总人群定义与优先级 |
| 跨人群对比矩阵 | 需求/场景/价格敏感度/渠道/战略吸引力 | 快速横向比较与决策 |
| 下游交接包 | for_1_4_trend_analysis / for_1_5_opportunity_analysis / for_2_1_market_positioning | 模块化交接 |

### 4.3 数据来源标注规范

- **高可信**：完整来源七字段（类型/标题/机构/发布时间/时效/获取渠道/链接）
- **中可信**：机构 + 标题 + 日期 + 时效
- **低可信**：来源 + 原因说明
- **信息不足**：统一标注 `unknown` 或 `insufficient evidence`

## 五、分析规则与决策标准

### 5.1 主轴选择：二元检查清单（0/1）

> 说明：本清单用于**主轴选择评分**，与第六章P0/P1/P2质量检查清单不同。

#### A. 决策相关性（5项，每项0/1）
- [ ] A1 该主轴可直接区分不同策略动作（定位/产品/渠道/传播）
- [ ] A2 该主轴可解释主要转化差异或复购差异
- [ ] A3 该主轴与研究目的（4选项之一）直接相关
- [ ] A4 该主轴可形成可执行的人群优先级排序
- [ ] A5 该主轴能稳定映射到下游接口字段

#### B. 品类购买逻辑（3项，每项0/1）
- [ ] B1 该主轴符合该品类真实决策机制（而非泛化标签）
- [ ] B2 该主轴可解释购买路径中的关键节点差异
- [ ] B3 该主轴可与1.1/1.2输出逻辑一致联动

#### C. 数据可获取性（2项，每项0/1）
- [ ] C1 该主轴有可验证外部或内部证据支撑
- [ ] C2 该主轴可在当前项目时限内稳定补充证据

**加权计算公式（强制）**：  
`总分 = (A/5)×40 + (B/3)×35 + (C/2)×25`

### 5.2 Tie-break 规则（强制）

1. 分差 < 1 分 → 启用**混合模式**（主轴+次轴并行定义）。
2. 完全平分 → 按以下顺序裁决：
   - 决策相关性优先
   - 品类逻辑优先
   - 简化主轴（可解释性更高、执行成本更低）

### 5.3 强制覆盖层（所有主轴都必须叠加）

- 价格敏感度
- 品牌忠诚度
- 渠道偏好

### 5.4 人群识别流程（标准模式3-5类）

1. 先按主轴切分候选人群
2. 再叠加强制覆盖层修正边界
3. 执行互斥性检查（重叠需合并/拆分说明）
4. 输出3-5类（标准模式）；过少或过多按边界规则处理

### 5.5 11维度明细（8必须 + 3可选/触发，附录C一致）

#### 必须维度（8 项）
| # | 维度 | 说明 | 数据来源 |
|---|------|------|---------|
| 1 | 人群细分（主轴+覆盖层） | 3-5 类人群，每类有定义层+描述层+决策层 | 主轴选择评分 |
| 2 | 购买动机 | 功能需求 vs 情感驱动 vs 社交需求 | 查询模板 #2 |
| 3 | 消费行为 | 购买频率、单次金额、渠道偏好 | 查询模板 #10, #6 |
| 4 | 决策路径 | 认知→考虑→购买→复购的关键节点 | 查询模板 #4 |
| 5 | 价格敏感度 | 价格弹性、溢价接受度、促销敏感度 | 查询模板 #5 |
| 6 | 品牌忠诚度 | 忠诚度分层、切换成本、复购率 | 查询模板 #7 |
| 7 | 消费场景 | 使用场景、购买场景 | 查询模板 #3 |
| 8 | 人群画像 | 年龄、收入、地域、职业 | 查询模板 #1 |

#### 可选/触发维度（3 项）
| # | 维度 | 触发条件 | 数据来源 |
|---|------|---------|---------|
| 9 | 购后满意度/NPS | 1.2 口碑数据可用（竞品有≥50 条评价） | 1.2 C1 输出 |
| 10 | 价值观/生活方式 | 品类与生活方式强相关（如奢侈品、运动品牌） | 查询模板 #1 + #8 |
| 11 | 媒体接触习惯 | 所有品类默认搜索，但仅在有明确数据时填充 | 查询模板 #8 |

### 5.6 证据门控规则（强制）

- 有证据：填写字段 + 来源标注 + 置信度
- 无证据：明确标注 `unknown` 或 `insufficient evidence`
- 禁止：无证据情况下补全画像字段、推断具体数值

### 5.7 数据融合规则（强制）

**核心原则**：信任1.1/1.2 + 逻辑一致性检查 + 冲突处理协议

1. **信任链**：默认信任1.1/1.2输出，不做重复审计式重建。
2. **一致性检查**：若本模块新证据与上游冲突，检查口径/时效/样本差异。
3. **冲突处理协议**：
   - 差异<5%：标注"已验证"，可融合
   - 差异5-10%：保留上游值为主，标注差异
   - 差异>10%：标注冲突并双值并存，触发用户决策或人工复核
4. **口碑可用性条件**：不假设1.2口碑数据可用；不可用时保持字段存在并标注不可用状态。

### 5.8 降级模式（无外部数据）

触发条件：外部数据源调用失败或高可信来源不足。  
执行方式：
1. 仅使用用户输入 + 1.1/1.2 + 内部文件
2. 输出更少且更宽泛的人群（建议2-3类）
3. 全面降低置信度并标注"不用于高风险下游决策"
4. 保留待补证据清单，后续可升级为标准模式

### 5.9 B2B 适配说明

- "消费者"映射为"购买决策单元（DMU）"
- 人口统计映射为企业特征/角色（行业、规模、岗位、权限）
- 媒体接触映射为信息来源/专业渠道（行业媒体、展会、专家网络）
- 购买路径映射为组织决策流程（需求提出→评估→审批→采购→复购）

### 5.10 前置缺失与人群合并/拆分边界规则

1. **1.1 缺失**：视为前置条件失败，停止正式执行，并要求先完成 1.1；不得生成正式人群细分或下游交接。  
2. **1.2 缺失**：默认停止正式流程；仅在用户明确要求继续时，方可进入**有限探索模式**：
   - 输入仅限用户输入 + 1.1 + 内部文件 + 可获得的少量外部证据
   - 输出仅供探索，**不用于下游决策**，且不生成正式 downstream_handoff
   - 所有人群置信度统一降为 `low`，并附"待1.2补齐后重做"说明
3. **主轴自然仅产出 2 类人群**：标准模式下先引入次轴或覆盖层再做一轮拆分；若仍仅 2 类，则仅可作为精简/降级输出保留，并标注"不适用于细粒度下游分配"。  
4. **主轴自然产出 6+ 类人群**：先按决策相关性、互斥性、证据充分度合并相近人群；合并后仍超过 5 类时，仅保留 3-5 类核心人群，其余并为"边缘/长尾人群"并记录合并依据。  
5. **合并/拆分记录要求**：必须记录触发原因、采用规则、受影响字段，以及对下游交接的影响。

## 六、质量检查清单（P0/P1/P2）

### P0 级（阻塞项 — 最小可用且决策安全）
- [ ] 主轴评分矩阵已完成，胜者与次胜者分差已记录（分差<1 分时已触发混合模式）
- [ ] 主轴 + 强制覆盖层已明确声明，且所有人群一致应用
- [ ] 至少识别 3 类人群（标准模式目标 3-5 类）
- [ ] 人群互斥性检查通过：无两人群判定条件实质重叠（如有重叠已标注合并/拆分说明）
- [ ] 每个人群卡片包含完整定义层（ID/名称/主轴定位/判定条件/排除条件/规模/增速）
- [ ] 核心层字段已存在，每个字段已填证据或明确标注"证据不足"
- [ ] 置信度标签已填写（人群存在性 + 每个非定义层字段）
- [ ] 1.1/1.2 数据已正确引用，含 1.2 口碑数据可用/不可用状态
- [ ] 最小跨人群对比表已生成（覆盖需求/场景/价格敏感度/渠道/战略吸引力）
- [ ] 下游交接 JSON 已生成且符合 Schema
- [ ] 数据时效检查通过或已标注过期警告

### P1 级（警告项 — 实质性提升战略价值）
- [ ] 人群数量 3-5 类（标准模式）；如有例外已标注原因
- [ ] 决策层已完整输出（定位/产品/定价/渠道/传播启示）
- [ ] 优先级排序包含至少 2 个明确依据（如吸引力+战略匹配度）
- [ ] 每个人群的竞争覆盖度已评估（过度/充分/不足/空白）
- [ ] 人群集合通过覆盖合理性检查：主要需求模式均已覆盖或明确排除
- [ ] AI 推导结论与证据-backed 观察已明确区分
- [ ] 跨源一致性已检查；重大矛盾已标注而非平滑处理

### P2 级（优化项 — 提升丰富度和可信度）
- [ ] 扩展层字段填充率 >50%
- [ ] 每个高优先级人群包含至少 1 条具体消费者语录/评价摘录/案例（如有数据）
- [ ] 高影响力声明使用至少 1 个高可信来源（如有可用来源）
- [ ] 高可信来源占比目标按品类调整，非固定 >50%
- [ ] 来源多样性存在（如平台评价 + 搜索证据 + 报告/社区数据）

**通过规则**：
1. P0全部通过：可提交
2. P1完成度<80%：需添加未完成项提示
3. P0未通过：停止交付并返回阻塞项
